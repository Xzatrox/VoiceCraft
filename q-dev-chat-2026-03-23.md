## Key Insights
- **Project Structure**: VoiceCraft is an Electron + React (TypeScript) app for converting books to audio using multiple TTS providers (Silero, Coqui, Piper, RHVoice, ElevenLabs, Qwen). Frontend in `src/`, Electron backend in `electron/`.
- **Windows shell:true pitfall**: Node.js `spawn` with `shell: true` on Windows passes args through `cmd.exe`, which interprets `>`, `<`, `|`, `&` as shell operators. Package specifiers with version constraints must be quoted.
- **i18n**: App supports English (`en.ts`) and Russian (`ru.ts`) with types in `types.ts`. All translation keys must be added to all three files.
- **TTS Server Architecture**: Python Flask server (`tts_server.py`, generated at runtime by `getTTSServerScriptContent()`) handles Silero, Coqui, and Qwen model loading/unloading/generation. Status endpoint returns loaded state for each engine. Server runs on port 5050.
- **Engine Python Isolation**: Each engine (silero, coqui, qwen) has its own Python installation under `{engine}-{accelerator}/python/`. The TTS server picks one Python to run with, preferring Coqui > Qwen > Silero.
- **Converter Pattern**: Both `convertToSpeech` and `previewVoice` in converter.ts build an `allVoices` array from all provider voice lists, find the voice by shortName, then dispatch to provider-specific generation functions via switch statements.
- **Type Declaration Duplication**: Types are declared in both `src/vite-env.d.ts` (for renderer process) and `electron/preload.ts` (for preload bridge). Both must be kept in sync when adding new providers.
